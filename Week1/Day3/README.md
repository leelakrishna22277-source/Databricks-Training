## 🚀 Topics Covered

Today I practiced advanced SQL concepts  which includes joins, aliases, subqueries, filtering, and pattern matching

---

## 📚 What I Learned

### 🔹 SQL EXISTS
- Used `EXISTS` to check whether a subquery returns records
- Learned how EXISTS works with correlated subqueries
- Understood how it improves conditional filtering

---

### 🔹 SQL Wildcards
- Practiced pattern matching using wildcards
- Used:
  - `%` → matches multiple characters
  - `_` → matches a single character
- Combined wildcards with `LIKE`

---

### 🔹 SQL DISTINCT
- Used `DISTINCT` to remove duplicate values
- Retrieved unique records from columns

---

### 🔹 SELECT TOP / LIMIT
- Learned how to retrieve limited rows
- Used `LIMIT` to fetch top records
- Practiced sorting before limiting results

---

### 🔹 SQL LIKE
- Used `LIKE` operator for pattern searching
- Filtered names based on starting, ending, and containing characters

---

### 🔹 SQL Alias
- Used aliases with `AS`
- Improved readability of query outputs
- Renamed columns temporarily

---

### 🔹 INNER JOIN
- Combined matching records from two tables
- Retrieved related data using common columns

---

### 🔹 LEFT JOIN
- Retrieved all records from left table
- Displayed NULL values for unmatched records

---

### 🔹 OUTER JOIN
- Learned concept of outer joins
- Understood how unmatched rows are included

---

### 🔹 SQL Nested Queries
- Used subqueries inside SELECT and WHERE
- Retrieved filtered results using inner queries
- Practiced nested aggregate conditions

## 💡 Key Takeaways

- SQL joins are essential for combining related tables
- Wildcards and LIKE are useful for searching patterns
- DISTINCT helps remove duplicate records
- Nested queries simplify complex filtering
- Aliases improve query readability
