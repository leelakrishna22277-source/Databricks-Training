Week 1 Day 3 - SQL Learning
===========================

Overview
--------

In Week 1 Day 3, I learned different SQL concepts used for retrieving, filtering, grouping, and analyzing data from database tables. I practiced multiple types of queries using employee data such as employee names, salaries, departments, cities, and experience. These topics helped me understand how SQL is used in real-world database operations.

Topics Learned
==============

1\. SELECT
----------

The SELECT statement is used to retrieve data from database tables. It is one of the most important SQL commands used for displaying records and columns.

### What I Learned

*   Display all employee details
    
*   Retrieve employee names and salaries
    
*   Display employee names with departments
    
*   Filter employee-related information
    
*   Retrieve selected columns only
    

### How It Is Used

*   To display complete table data
    
*   To retrieve specific employee information
    
*   To view department and salary details
    

2\. WHERE Clause
----------------

The WHERE clause is used to filter records based on conditions. It helps in retrieving only required data from the database.

### What I Learned

*   Filter employees based on salary
    
*   Retrieve employees from specific cities
    
*   Filter employees using experience
    
*   Retrieve employees from specific departments
    

### How It Is Used

*   To display employees with high salary
    
*   To retrieve employees from Hyderabad
    
*   To filter employees with less experience
    

3\. GROUP BY
------------

GROUP BY is used to group rows with similar values together. It is mainly used with aggregate functions.

### What I Learned

*   Group salary department-wise
    
*   Count employees city-wise
    
*   Find average salary in departments
    
*   Analyze grouped employee data
    

### How It Is Used

*   To generate department reports
    
*   To analyze employee statistics
    
*   To summarize data based on groups
    

4\. HAVING Clause
-----------------

The HAVING clause is used to filter grouped records after applying GROUP BY.

### What I Learned

*   Filter departments with more employees
    
*   Find departments with high average salary
    
*   Filter grouped city records
    
*   Analyze grouped salary conditions
    

### How It Is Used

*   To filter grouped data
    
*   To apply conditions on aggregate results
    
*   To generate advanced reports
    

5\. TOP Clause
--------------

The TOP clause is used to limit the number of records returned in the output.

### What I Learned

*   Retrieve highest paid employees
    
*   Display employees with highest experience
    
*   Retrieve top salary records
    
*   Limit result rows
    

### How It Is Used

*   To display top employee records
    
*   To retrieve limited data
    
*   To analyze highest values
    

6\. DISTINCT Keyword
--------------------

DISTINCT is used to display only unique values by removing duplicates.

### What I Learned

*   Retrieve unique departments
    
*   Display unique cities
    
*   Display unique salary values
    
*   Retrieve unique combinations of columns
    

### How It Is Used

*   To avoid duplicate records
    
*   To display unique data values
    
*   To simplify reports
    

7\. Comparison Operators
------------------------

Comparison operators are used to compare values in conditions.

### What I Learned

*   Compare salaries and experience
    
*   Retrieve employees using conditions
    
*   Use greater than, less than, and equal conditions
    

### How It Is Used

*   To filter employees with high salary
    
*   To compare employee experience
    
*   To retrieve matching records
    

8\. Logical Operators
---------------------

Logical operators are used to combine multiple conditions in queries.

### What I Learned

*   Use AND operator
    
*   Use OR operator
    
*   Use NOT operator
    
*   Apply multiple conditions together
    

### How It Is Used

*   To filter employees based on multiple conditions
    
*   To retrieve employees from multiple cities
    
*   To exclude specific departments
    

9\. IN and NOT IN Operators
---------------------------

The IN operator is used to match multiple values, while NOT IN is used to exclude values.

### What I Learned

*   Retrieve employees from selected cities
    
*   Filter departments using multiple values
    
*   Exclude cities and departments
    

### How It Is Used

*   To simplify multiple conditions
    
*   To retrieve matching records
    
*   To exclude unwanted values
    

10\. BETWEEN Operator
---------------------

BETWEEN is used to retrieve values within a specific range.

### What I Learned

*   Filter salary ranges
    
*   Retrieve employees based on experience range
    
*   Retrieve employee IDs within limits
    

### How It Is Used

*   To display records between ranges
    
*   To filter numerical data
    
*   To simplify range conditions
    

11\. LIKE Operator
------------------

The LIKE operator is used for pattern matching in SQL.

### What I Learned

*   Find names starting with specific letters
    
*   Find names ending with specific letters
    
*   Search values containing characters
    
*   Perform text-based searching
    

### How It Is Used

*   To search employee names
    
*   To filter city and department names
    
*   To perform pattern matching operations
    

Conclusion
==========

In Week 1 Day 3, I learned important SQL concepts related to data retrieval, filtering, grouping, aggregation, and pattern matching. I practiced working with employee-related data using multiple SQL clauses and operators. These topics improved my understanding of SQL queries and helped me build a strong foundation in database operations and data analysis.
